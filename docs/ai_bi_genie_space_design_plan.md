# AI/BI Genie Space Design Plan

This document is a Databricks-implementation design for adding an AI/BI Genie
Space to the AdventureWorks BI project. It is written for a Databricks Genie
coding assistant or Databricks coding assistant to implement directly inside
Databricks.

## Assumptions

- The Lakeflow pipeline has already materialized the BI tables in Unity
  Catalog.
- The deployment target uses catalog `adventure_works_dw` and a target schema
  supplied by the current Databricks Asset Bundle target.
- The existing dashboard bundle resource points to warehouse
  `a10e62df4c510ef3`.
- The Genie Space should answer questions from the curated BI layer only, not
  from raw bronze, silver, gold fact, or gold dimension tables.
- The first production-ready scope is Internet Sales performance. Keep this
  space focused instead of making it a general AdventureWorks assistant.

## Databricks Guidance Applied

This plan follows the official Databricks AI/BI and Genie guidance:

- AI/BI combines dashboards, Genie Spaces, and business semantics for self
  service analytics.
- Genie Spaces work best when they are domain-specific and curated by analysts
  with well documented Unity Catalog tables, business terms, example SQL, and
  focused instructions.
- Databricks recommends starting small and staying focused. A Genie Space can
  support up to 30 tables or views, but the best-practice target is five or
  fewer tables.
- Example SQL queries and SQL expressions should carry most of the semantic
  teaching. Plain text instructions should be concise and used only where the
  structured semantic tools are not enough.
- Published AI/BI dashboards can include an autogenerated companion Genie Space.
  Dashboard filters carry into the autogenerated companion space, but not into a
  separately linked existing Genie Space.
- Genie can answer from the full dashboard dataset results, so dashboard datasets
  must exclude sensitive or confusing fields that viewers should not query.
- Space quality should be validated with benchmark questions, generated SQL
  inspection, user feedback, and monitor-tab review.

## Recommended Implementation Path

Implement this in two layers.

### Layer 1: Dashboard Companion Genie Space

Use the existing AI/BI dashboard as the first Genie entry point.

When publishing the `AdventureWorks Executive Sales Dashboard`, keep the
dashboard Genie option enabled so Databricks creates the autogenerated companion
Genie Space. This gives the most consistent dashboard experience because
dashboard filters are included in chat context for autogenerated companion
spaces.

This layer is the default recommendation for dashboard viewers.

### Layer 2: Curated Standalone Genie Space

Create a separate curated space named:

```text
AdventureWorks Internet Sales Genie Space
```

Use this standalone space for broader natural-language analysis over the BI
tables. Link it to the dashboard only if you explicitly accept the Databricks
behavior that dashboard filters do not carry into linked existing spaces.

This layer is the default recommendation for analyst exploration and semantic
curation.

## Data Scope

Set the target context before configuring or testing:

```sql
USE CATALOG adventure_works_dw;
USE SCHEMA <target_schema>;
```

Use exactly these BI-facing tables:

- `bi_sales_order_line`
- `bi_sales_monthly`
- `bi_product_performance`
- `bi_territory_performance`
- `bi_customer_segments`

Do not add lower-level source tables to the Genie Space. These five tables keep
the space inside Databricks' recommended focused-table range and preserve the BI
layer's curated labels and metric definitions.

## Space Description

Use this description for the standalone space:

```text
Answers Internet Sales questions for the AdventureWorks BI layer. Use this
space for executive revenue, order, discount, product, territory, and customer
segment analysis based on curated BI tables only.
```

## Permissions And Prerequisites

Before implementation, verify these requirements in Databricks:

- Partner-powered AI features are enabled for the account and workspace.
- The selected warehouse is pro or serverless SQL.
- The space creator has `CAN USE` on the warehouse.
- The space creator has `SELECT` on all five BI tables.
- End users have Databricks SQL entitlement.
- End users have `SELECT` on all Unity Catalog objects used by the space.
- End users have `CAN VIEW` or `CAN RUN` on the Genie Space.
- Workspace users who administer the space have `CAN EDIT` or `CAN MANAGE`.

Use embedded compute credentials for the space warehouse, but rely on Unity
Catalog permissions for end-user data access.

## Business Semantics

Add these SQL expressions or equivalent business semantics to the Genie Space.
Use the names below as the canonical terms Genie should understand.

### Measures

| Business term | SQL expression |
| --- | --- |
| Net Sales | `sum(net_sales_amount)` |
| Gross Sales | `sum(gross_sales_amount)` |
| Discount Amount | `sum(discount_amount)` |
| Discount Rate | `sum(discount_amount) / nullif(sum(gross_sales_amount), 0)` |
| Order Quantity | `sum(order_quantity)` |
| Order Count | `sum(order_count)` on aggregate tables; `count(DISTINCT sales_order_id)` on `bi_sales_order_line` |
| Customer Count | `sum(customer_count)` on aggregate tables when the grain is compatible; `count(DISTINCT customer_id)` on `bi_sales_order_line` for true distinct customers |
| Average Order Value | `sum(net_sales_amount) / nullif(sum(order_count), 0)` |
| Average Selling Price | `sum(net_sales_amount) / nullif(sum(order_quantity), 0)` |

### Dimensions

| Business term | Preferred field |
| --- | --- |
| Month | `order_year`, `order_month`, `order_month_name` |
| Product Category | `product_category_name` |
| Product Subcategory | `product_subcategory_name` |
| Product | `product_name` |
| Territory Group | `territory_group` |
| Territory | `territory_name` |
| Country | `territory_country_region_code` |
| Income Segment | `yearly_income` |
| Education Segment | `education` |
| Occupation Segment | `occupation` |

### Filters

| Business term | SQL guidance |
| --- | --- |
| Discounted sales | `discount_amount > 0` |
| High discount rate | `discount_rate >= 0.15` when `discount_rate` exists, otherwise compute from sales amounts |
| Top products | Order by `sum(net_sales_amount) DESC` |
| Top territories | Order by `sum(net_sales_amount) DESC` |
| Monthly trend | Group by `order_year`, `order_month`, and `order_month_name`; sort by year then month |

## General Genie Instructions

Use these as the concise general instructions for the standalone space:

```text
Answer questions using only the five curated BI tables in this space. Treat
"sales" and "revenue" as Net Sales unless the user explicitly asks for Gross
Sales. Prefer aggregate BI tables for monthly, product, territory, and customer
segment questions. Use bi_sales_order_line only when the user asks for line
detail, distinct customer counts, or a grain not available in the aggregate
tables. For monthly trends, sort by order_year and order_month. Format currency
amounts as monetary values, counts as whole numbers, and rates as percentages.
If the user asks an ambiguous question without a time range, answer for all
available data and state that no date filter was applied. Ask a clarification
question when the requested metric, time period, or grouping cannot be inferred.
Do not query bronze, silver, gold fact, or gold dimension tables.
```

## Clarification Rules

Configure clarification behavior for these cases:

| Trigger | Missing detail | Action |
| --- | --- | --- |
| "best" or "top" without metric | Ranking metric | Ask whether to rank by Net Sales, Order Count, Order Quantity, or Customer Count. |
| "growth" without comparison period | Time comparison | Ask whether to compare month over month, year over year, or full-year totals. |
| "customers" without desired meaning | Customer metric | Ask whether the user wants distinct customers or customer segment performance. |
| "discount impact" without output preference | Metric or dimension | Ask whether to show Discount Amount, Discount Rate, or Net Sales after discount. |

## Example SQL Queries

Create example SQL queries in the Genie Space with natural question titles.
Use these examples to teach prompt patterns, not just exact wording.

### Total Executive KPIs

Question title:

```text
What are total net sales, gross sales, discounts, orders, customers, and average order value?
```

SQL:

```sql
WITH sales_totals AS (
  SELECT
    sum(net_sales_amount) AS net_sales_amount,
    sum(gross_sales_amount) AS gross_sales_amount,
    sum(discount_amount) AS discount_amount,
    sum(order_quantity) AS order_quantity,
    sum(order_count) AS order_count
  FROM bi_sales_monthly
),
customer_totals AS (
  SELECT count(DISTINCT customer_id) AS customer_count
  FROM bi_sales_order_line
)
SELECT
  sales.net_sales_amount,
  sales.gross_sales_amount,
  sales.discount_amount,
  sales.order_quantity,
  sales.order_count,
  customers.customer_count,
  sales.net_sales_amount / nullif(sales.order_count, 0) AS average_order_value,
  sales.discount_amount / nullif(sales.gross_sales_amount, 0) AS discount_rate
FROM sales_totals AS sales
CROSS JOIN customer_totals AS customers;
```

### Monthly Sales Trend

Question title:

```text
How are net sales, gross sales, discounts, orders, and customers trending by month?
```

SQL:

```sql
SELECT
  order_year,
  order_month,
  order_month_name,
  net_sales_amount,
  gross_sales_amount,
  discount_amount,
  order_count,
  customer_count,
  discount_amount / nullif(gross_sales_amount, 0) AS discount_rate
FROM bi_sales_monthly
ORDER BY order_year, order_month;
```

### Product Category Ranking

Question title:

```text
Which product categories have the highest net sales?
```

SQL:

```sql
SELECT
  product_category_name,
  sum(net_sales_amount) AS net_sales_amount,
  sum(gross_sales_amount) AS gross_sales_amount,
  sum(discount_amount) AS discount_amount,
  sum(order_quantity) AS order_quantity,
  sum(order_count) AS order_count,
  sum(discount_amount) / nullif(sum(gross_sales_amount), 0) AS discount_rate
FROM bi_product_performance
GROUP BY product_category_name
ORDER BY net_sales_amount DESC;
```

### Top Products

Question title:

```text
What are the top 25 products by net sales?
```

SQL:

```sql
SELECT
  product_name,
  product_category_name,
  product_subcategory_name,
  net_sales_amount,
  gross_sales_amount,
  discount_amount,
  discount_rate,
  order_quantity,
  order_count,
  customer_count
FROM bi_product_performance
ORDER BY net_sales_amount DESC
LIMIT 25;
```

### Territory Performance

```text
Which territories and territory groups perform best by net sales?
```

SQL:

```sql
  territory_name,
  territory_country_region_code,

  discount_rate,

```

### Customer Segment Performance

Question title:

```text
How do sales compare by customer income, education, and occupation?
```

SQL:

```sql
SELECT
  yearly_income,
  education,
  occupation,
  sum(net_sales_amount) AS net_sales_amount,
  sum(gross_sales_amount) AS gross_sales_amount,
  sum(discount_amount) AS discount_amount,
  sum(order_quantity) AS order_quantity,
  sum(order_count) AS order_count,
  sum(customer_count) AS customer_count
FROM bi_customer_segments
GROUP BY yearly_income, education, occupation
ORDER BY net_sales_amount DESC;
```

### High Discount Products

Question title:

```text
Which products have the highest discount rates?
```

SQL:

```sql
SELECT
  product_name,
  product_category_name,
  product_subcategory_name,
  net_sales_amount,
  gross_sales_amount,
  discount_amount,
  discount_rate,
  order_quantity,
  order_count
FROM bi_product_performance
WHERE gross_sales_amount > 0
ORDER BY discount_rate DESC, discount_amount DESC
LIMIT 25;
```

### Territory Discount Comparison

Question title:

```text
Where are discounts highest by territory?
```

SQL:

```sql
  territory_group,
  territory_name,
  territory_country_region_code,
  net_sales_amount,
  gross_sales_amount,
  discount_amount,
  discount_rate,
  order_count
FROM bi_territory_performance
WHERE gross_sales_amount > 0
ORDER BY discount_rate DESC, discount_amount DESC;
```

## Benchmark Questions

Create a benchmark set before sharing the space broadly. For each benchmark,
provide the expected SQL answer in the benchmark configuration when the UI asks
for it.

Use these questions and add two to four alternate phrasings for each:

- What are total net sales, orders, customers, discount rate, and average order
  value?
- Show monthly net sales and order count over time.
- Which product categories generate the most net sales?
- What are the top 25 products by net sales?
- Which territories have the highest net sales?
- Which territories have the highest discount rate?
- Which customer income segments generate the most sales?
- Compare sales by education and occupation.
- What is the average selling price by product category?
- Which products have high discounts but low net sales?

Success target:

- Core KPI and ranking questions produce correct SQL and results.
- Generated SQL uses only the five BI tables.
- Generated SQL uses the documented metric definitions.
- Ambiguous questions trigger clarification when the missing detail changes the
  answer materially.

## Bundle And Versioning Plan

Use Databricks Asset Bundles for versioned deployment where supported.

Implementation guidance:

- Keep the existing dashboard bundle resource for the AI/BI dashboard.
- If the current Databricks bundle schema supports a Genie Space resource, add a
  focused `resources/genie_spaces.yml` file for the standalone space.
- If the bundle schema in the target workspace does not support Genie Space
  resources yet, keep this document as the source of truth and implement the
  space through the Databricks UI or supported APIs.

  variables already used by the project.
- Do not duplicate the BI model logic inside Genie configuration. Point Genie at
  the existing BI tables and teach metrics through expressions and examples.
