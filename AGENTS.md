# Project Working Agreement

This file applies recursively to the entire repository. All contributors and
agents must follow it unless a more specific `AGENTS.md` exists in a
subdirectory.

## Git Management
- Don't commit your changes unless explicitly told to
- when you push your changes, make sure to check the CI runs successfully before giving task completion confirmation to user


## Databricks interactions
- Use databricks profile named "personal" to access databricks resources related to this project

## Documentation
- When you add new code, add docstring where it makes sense, no over-engineering.
- When you make any change, make sure to add/update related documentations
